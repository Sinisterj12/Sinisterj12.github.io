About

Operator-led systems builder

I build operational software from the inside of the problem. My work is shaped by hands-on support leadership, retail POS infrastructure, field troubleshooting, PCI-aware environments, and the kind of workflow friction that only shows up when real users are under pressure.

My edge is combining domain knowledge with AI-assisted development: I define the workflow, pressure-test the output, and use AI coding agents to move faster while still demanding runtime checks, clean handoff, and proof that the system actually works. The result is not demo software. It is practical tooling built around speed, context, reliability, and operator trust.

Active Internal Platform CRM Wrapper + Operations Hub

RDSCore CRM Workspace

RDSCore is a private CRM wrapper and operations workspace built to make support work faster than the legacy CRM experience. Instead of forcing the team through slow clicks and scattered screens, the goal is one fast search surface that can pull together store context, ticket history, contacts, vendor references, employee visibility, repeat-issue warnings, and operational notes.

Under the hood, the system is being shaped around a FastAPI backend, NetworkX relationship graph, structured search routing, runtime contracts, and selective AI escalation. The point is not to make a flashy dashboard. The point is to give the helpdesk a cleaner command surface that turns messy operational data into something searchable, connected, and actually usable during a busy support day.

Private Repository After-Hours Alerting

After Hours Emergency Alert System

One of the first tools I built with AI. I run a helpdesk where after-hours calls route to the on-call tech, but voicemail alert emails were often handled without anyone replying "got it" for the team to see. I built a Google Apps Script workflow that watches for those conditions and alerts a shared Google Chat room so the team knows a call may still need attention. It gave us a practical way to catch misses, confirm accountability, and step in fast when needed.

Private Repository Disk Space Recovery

GCleaner

Built to recover hard drive space on grocery store registers before they hit the point of failure. Support techs were losing time logging in, hunting through systems, and sometimes missing the cleanup locations that mattered most. I built a discovery-first tool that shows what can be cleaned before the real cleanup script runs, making the process faster, safer, and more consistent. The remaining problem is Windows installer bloat, which needs a separate tool and is already on my list.

Private Repository Sheets OCR Workflow

Sales Order OCR Extraction

A Google Sheets and Apps Script workflow I built to extract sales orders from two completely different source formats and turn them into a clean, structured output. From a custom menu in Sheets, I can choose the correct extraction path for each order source, then export the results into a template-driven sheet formatted exactly how our install process needs it. It is still being stabilized, but the larger goal is to evolve it into a more automatic workflow that also feeds asset tracking, inventory visibility, and CRM build-out.
